Introduction to Parallel Projection CT Reconstruction [00:00:03]

Welcome to the second video of the CT Reconstruction tutorial series. This tutorial will demonstrate how to use the CT Reconstruction plugin for parallel projection data reconstruction. While we covered parallel projection data reconstruction in the previous video, this tutorial will focus on the specific details that differ from the previous method.

Importing Projection Data [00:00:44]

First, let's import our projection data. In the previous tutorial, we used a series of TIFF images, but there's a simpler approach:

  • If you've previously saved an OIS object, you can directly import it by clicking the import button
  • This eliminates the need to import all TIFF files again
  • Select the required data from the saved OIS object

Our sample dataset has the following characteristics:
- Resolution: 2560×400
- Depth (number of projections): 1025
- Angular range: 0 to 180 degrees
- Content: Ceramic composite material with fibers

This data comes from a synchronized loop obtained from a micro-CT beamline of an accelerator, making it a parallel beam dataset. Special thanks to Ali Adrian from Colorado University for providing this projection data.

As mentioned in the previous video, ensure that the detector plan is on the XY plan with its rotation axis in the outward direction.

Setting Up the CT Reconstruction Workflow [00:02:58]

  1. Open the CT reconstruction test board
  2. Choose the imported data as the projection data
  3. In the collection parameters, select Parallel Beam

For parallel beam reconstruction, we need to configure five parameters:

  • Smallest angle: Usually 0
  • Angular range: 180 degrees in our example
  • Number of projections: 1025 images
  • Detector pixel size: X and Y values are typically set to default (1) for parallel beam data, as equipment manufacturers often don't provide this information
  • Rotation center offset: Input the known value or use the "Calculate Rotation Center" button to determine the offset (this will be explained in the third video)

Selecting a Reconstruction Engine [00:04:33]

There are several reconstruction engine options available:

RTK and FTK

These engines include a parameter called "mask ratio" which determines how much of the surrounding data will be removed after reconstruction:
- Mask ratio of 1: No masking applied
- Mask ratio of 0: All data masked
- Values between 0 and 1: Creates a circular mask with the specified ratio of the full diameter

TomoPi

TomoPi offers two algorithm options:
- Grid Rec: Requires selecting a tracking pen and filter (default is SHAPE)
- RTK: An iterative method requiring multiple iterations

TomoPi's Grid Rec is typically faster than RTK and is often the preferred choice.

Performing the Reconstruction [00:06:34]

After setting all parameters, proceed with a calculation preview:

  1. Click the preview button to generate a preliminary reconstruction
  2. Observe the circular mask applied to the data (result of the mask ratio setting)
  3. Examine the quality of the reconstruction - you should be able to see the fibers and composite material structure

You can adjust the mask ratio to include more or less of the surrounding data. For example, reducing it to 0.8 or 0.7 will create a smaller circular region of interest.

Different reconstruction engines may produce slightly different results in terms of scale and gray values. You can compare the output from TomoPi Grid Rec and RTK to determine which works best for your data.

Final Reconstruction [00:08:37]

Once you're satisfied with the preview and collection parameters:

  1. Click "Reconstruct to current workspace"
  2. Wait for the reconstruction process to complete
  3. Examine the final reconstructed data

The reconstructed volume will show the fibers clearly in the middle of the sample. You can view the data from different angles to observe the horizontal and diagonal surfaces of the fibers.

Conclusion [00:09:50]

This tutorial demonstrated how to perform parallel projection CT reconstruction using the plugin. The process is similar to fan beam reconstruction with some specific parameter differences. The mask ratio feature provides a useful way to focus on the region of interest in your reconstruction.
