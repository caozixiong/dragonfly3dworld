ORS launches Dragonfly VizServer for simple and productive remote-access scientific visualization - (English)

Dragonfly Viz Server is a new solution that allows users to access and run Dragonfly scientific imaging software remotely from any device, freeing researchers from being tied to specific workstations while providing centralized management of resources, session continuity, and the flexibility to work from any location.

Introducing Dragonfly Viz Server: Freedom to Work Where You Want

Introduction [00:00:02]

Dragonfly Viz Server is a new way to run Dragonfly that creates a pattern for users allowing Dragonfly to meet them where they need to be, enabling scientific image processing with the freedom to work where and when you want.

The Problem with Traditional Scientific Imaging Resources [00:01:08]

Scientific imaging resources are traditionally allocated in a restrictive way:

  1. Initially, organizations invest in a single workstation and software license
  2. This workstation is placed in a lab or designated computer room
  3. As demand grows, multiple scientists need access but must share limited resources
  4. Organizations add more workstations, creating a "Viz Lab" or data analysis suite
  5. These Viz Labs are often tucked away in basements or hallways, far from scientists' preferred workspaces

This traditional model creates several pain points:
- Limited availability when computers are booked by other users
- Data accessibility issues when your data is on an unavailable workstation
- Software version inconsistencies across workstations
- Inconvenient locations away from scientists' offices or preferred work areas

Introducing Dragonfly Viz Server [00:05:33]

Dragonfly Viz Server offers a new pattern for using Dragonfly that provides freedom to work where you want. The solution:

  • Dragonfly installed and maintained in a centralized location (on-premises or cloud-based)
  • Users access and run Dragonfly remotely from anywhere
  • Compatible with various devices (laptops, workstations, even MacBooks or Chromebooks)
  • Centralized resource management and maintenance

How Dragonfly Viz Server Works [00:06:48]

  1. A user launches Dragonfly from their computer
  2. The Viz Server service automatically finds an available computer
  3. A connection is established within seconds
  4. The Dragonfly splash screen appears and the user can begin working
  5. Multiple users can connect simultaneously to different server instances
  6. Session continuation allows users to reconnect to their existing session from different locations

Frequently Asked Questions [00:18:13]

Overview Questions

Does this change anything with Dragonfly Workstation?
No. Both Dragonfly Workstation and Viz Server will continue to receive regular updates on the same schedule.

Does Viz Server have the same features as Workstation?
Yes, all the same features for image segmentation, visualization, animation, and analysis are available.

How much does Viz Server cost?
Contact sales for pricing plans tailored to your group size and institution type.

Does my non-commercial license include Viz Server?
No, Viz Server requires a separate license, though pricing plans are available for universities and non-commercial research institutes.

Can I upgrade my existing workstation license to run Viz Server?
Yes, contact sales to identify a migration plan.

Can Viz Server hardware be used for other software?
Yes, though it's not recommended as it may lead to under-provisioned hardware.

Client Configuration and Behavior

What are the computer requirements to access Viz Server?
Any computer with a keyboard, mouse, monitor, and network connection will work. Recommended: mouse with scroll wheel and monitor with 1920×1080 resolution or higher. Compatible with Windows, Linux, macOS, and Chrome OS.

Do I have to be in the same location as the Viz Server?
No, you only need a low-latency network connection. Users can connect from different cities or even continents.

Will Viz Server let me load or save data from my local computer?
Yes, though the experience is better when used with Project Organizer. Loading and saving files directly depends on network bandwidth.

What happens when all Viz Server nodes are in use?
Users will receive a message indicating all resources are in use and can try again later.

Server Configuration and Performance

Where is Viz Server hosted?
Viz Server is designed to be hosted and administered by the customer, either on-premises or in the cloud.

How is Viz Server different from Dragonfly Cloud?
Viz Server introduces session continuation and automatic load balancing, integrating and improving upon the former Dragonfly Cloud product.

What are the hardware requirements for Viz Server?
Requirements depend on your analysis needs. RAM can range from 32GB to 512GB. Each server node requires a workstation-class or data center-class NVIDIA GPU.

Will ORS sell me hardware for Viz Server?
ORS provides guidance on specifications, but customers should purchase hardware from their trusted vendors.

Can I repurpose my current Dragonfly hardware for Viz Server?
Yes, though you may need to upgrade your operating system to Windows Server.

Can I use Viz Server with only one server node?
Yes, even with a single node, you gain the flexibility to access your hardware remotely.

Can I use my university's HPC cluster for Viz Server?
Possibly, depending on your university's HPC resource management. This requires discussion between your IT department and the ORS Viz Server support team.

Does Viz Server rely on virtualization?
No, but it can work with virtualized systems, bare metal systems, or a combination.

Do I need a dedicated system administrator?
The Viz Server support team will help with configuration and deployment, but having colleagues with IT experience is beneficial.
