Pre-trained Models in Dragonfly: A Case Study with Bone MicroCT Data

This video provides a case study on the use of pre-trained models within the Dragonfly software for bone microCT data segmentation. The presenter, Huber, discusses the benefits of transfer learning and demonstrates the application of pre-trained models to improve segmentation performance, reduce training time, and accelerate convergence.

Benefits of Pre-trained Models [00:04:24]

  • Improved Performance: Pre-trained models consistently outperform untrained models in Dice score metrics. [00:07:14]
  • Faster Convergence: Pre-trained models reach higher accuracy levels more quickly during training. [00:09:41]
  • Reduced Training Time: Leveraging pre-existing knowledge accelerates the training process. [00:04:32]

Step-by-Step Guide: Using Pre-trained Models in Dragonfly

  1. Access Deep Learning Tool: Navigate to the Deep Learning tool within Dragonfly. [00:15:40]
  2. Create a New Model: Initiate the creation of a new deep learning model. [00:15:49]
  3. Select Pre-trained Model: Choose a pre-trained model from the "Pre-trained by Dragonfly Team" section. Available models include:
    • U-Net with varying depths. [00:16:09]
    • Sensor3D. [00:16:15]
    • ORS 500k (trained on diverse scientific data). [00:05:04]
    • Bone 500k (trained on grayscale bone images). [00:05:11]
  4. Data Calibration (Optional): If using datasets from multiple sources, utilize the "Calibrate Intensity Scale" tool to normalize intensity values. This ensures consistency across datasets. [00:11:09]
  5. Training: Train the selected pre-trained model using your specific dataset for semantic segmentation. [00:04:24]

Data Calibration [00:10:57]

  • The "Calibrate Intensity Scale" tool is crucial when working with datasets from different sources to ensure consistent intensity distributions. [00:11:09]
  • This tool allows training deep learning models without requiring normalization of datasets, preserving the original image data. [00:11:44]
  • Users can define a custom scale (e.g., a "Centigrade" scale for bone density) or use a universal scale. [00:12:53]

Universal Bone Segmentation Model [00:16:47]

  • The presenter is developing a universal model for unsupervised bone segmentation across different animal species and voxel sizes. [00:16:47]
  • This model aims to streamline bone segmentation, allowing users to focus on scientific analysis rather than manual segmentation. [00:17:09]
  • Data calibration is essential for the universal model to ensure scalability across diverse datasets. [00:17:56]

Conclusion [00:19:11]

Pre-trained models offer significant advantages for bone microCT data segmentation in Dragonfly. Users are encouraged to explore different pre-trained models and provide feedback to further enhance their capabilities. The development of a universal bone segmentation model promises to simplify bone analysis workflows.

Discussion Points

  • Model Selection: The choice of pre-trained model depends on the specific application and dataset characteristics. [00:42:47]
  • Patch Size: In the experiments, a patch size of 128x128 was used. [00:48:16]
  • Validation: Consistent validation datasets were used across different training set sizes to ensure fair comparisons. [00:47:07]
  • Calibration Scales: The Bone 500k model is calibrated, while the ORS 500k model is not. [00:36:30]
