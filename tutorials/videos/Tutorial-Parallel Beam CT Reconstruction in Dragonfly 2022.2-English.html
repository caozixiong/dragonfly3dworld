Reconstructing Parallel Beam CT Data Using City Reconstruction Plugin in Dragonfly

Introduction [00:00:05]

This tutorial demonstrates how to reconstruct parallel beam CT data using the City Reconstruction plugin in Dragonfly. While the process shares similarities with cone beam reconstruction, this guide focuses on the specific differences between the two methods.

Loading the Parallel Beam Dataset [00:00:42]

First, load your parallel beam dataset. In this example, we'll use a fiber-reinforced ceramic composites dataset shared by Ali Bejran from the University of Toronto. This dataset was acquired on a single-chrome parallel beam's macro-seated beamline.

The example dataset has:
- 1025 projections
- 400 slices (cropped for this tutorial)

When viewing the dataset, ensure that:
- The detector plane is on the XY plane
- The rotation axis is along the Y axis

Starting the City Reconstruction Plugin [00:02:06]

  1. Navigate to the Workflows menu and start the City Reconstruction plugin
  2. Select your raw data as the projection dataset

Setting Geometry Parameters [00:02:29]

For parallel beam datasets, the geometry parameters differ from cone beam reconstruction:

  1. Select Parallel Beam as the beam type
  2. Set the minimum angle (typically 0)
  3. Set the angle step (for our dataset with 1025 projections over 180 degrees, the angle step is 0.1756)
  4. Set detector spacing along X and Y (typically 1)

Important note about rotation center offset:
- The rotation center offset is typically given in pixels
- To calculate real values, multiply the pixel value by the detector spacing
- For our dataset, the rotation center offset is -107 mm with detector spacing of 1 mm

Choosing Reconstruction Algorithm [00:04:22]

There are several algorithm options available:

RTK Options

  • Select the RTK-FDK algorithm
  • Set the mask ratio (default is 0.9)
  • A mask ratio of 1 means no mask is applied
  • Lower values crop the central cylinder region and remove edge background

ThermalPipe Reconstruction Engine

Two algorithm options are available:
1. Grade Rig
- Choose your preferred filter

  1. ART (Iterative Algorithm)
  2. Set the number of iterations
  3. Note: This method typically takes longer to process

For most reconstructions, RTK-FDK or ThermalPipe Grade Rig are recommended for their efficiency.

Pre-processing Options [00:05:55]

Typically for parallel beam datasets, flat field correction is required. Pre-processing filters will be covered in a separate tutorial.

Computing Preview and Reconstruction [00:06:24]

  1. Click Compute Preview to generate a preview of the reconstruction
  2. This process is very fast for parallel beam data

  3. The preview will show the effect of the mask ratio, cropping out background edges

  4. To perform the full reconstruction:

  5. If you closed the City Reconstruction plugin but want to use the same parameters, open the City Reconstruction beta version
  6. Use Import Inputs from Preview to restore your settings
  7. Click Reconstruct to process the full volume

Examining the Reconstruction Results [00:08:59]

After reconstruction is complete:
- All 400 slices will be reconstructed
- View the X-Z plane to see the fibers
- Examine the reconstruction quality

Note that without pre-processing filters like flat field correction, the reconstruction may show artifacts. In the XY plane, you might observe ring artifacts that can be improved with appropriate pre-processing.

Conclusion [00:09:48]

This tutorial covered the basic process for reconstructing parallel beam CT data using the City Reconstruction plugin in Dragonfly. The key differences from cone beam reconstruction include:
- Simpler geometry parameters
- Different algorithm options
- Specific pre-processing requirements

For optimal image quality, additional pre-processing filters should be applied to remove artifacts, which will be covered in a separate tutorial.
