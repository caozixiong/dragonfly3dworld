July 2022 Workshop Video 8: Machine learning segmentation of a 3-horned chameleon - (English)

This video demonstrates traditional machine learning segmentation techniques for a chameleon head dataset, showcasing how these methods can identify bone structures that are missed by conventional thresholding approaches despite not being as widely used as newer deep learning methods.

Tutorial Content

Introduction to Classical Machine Learning Segmentation [00:00:02]

This tutorial demonstrates classical machine learning segmentation techniques in Dragonfly. Unlike deep learning methods covered later in the series, this approach represents the traditional segmentation tool that was first developed. While not as widely used today, it remains quite impressive for certain applications.

Dataset Overview [00:00:35]

For this demonstration, we'll use a three-horned chameleon head dataset. The goal is to segment bone structures inside the chameleon head, which presents challenges for normal thresholding techniques:

  • When using manual thresholding, some dense parts appear acceptable
  • However, many thin bone structures are missed entirely
  • Some areas are over-segmented

This is precisely where machine learning can improve segmentation results by being trained to identify these challenging structures.

Creating Training Data [00:01:55]

For machine learning segmentation in Dragonfly, we need to provide ground truth labeling of pixels:

  1. Go to the Segment tab
  2. Create a new Multi-ROI with three classes:
  3. Name it "sparse data for training"
  4. Set up three classes: bone, air, and flesh
  5. Assign colors to each class:
  6. Purple for air
  7. Red for flesh
  8. Green for bone

Unlike deep learning tools that require labeling all pixels, this traditional machine learning approach only needs sparse labeling of some representative pixels.

Labeling Pixels [00:03:22]

  1. Use the Brush tool to label representative pixels for each class
  2. Label some bone pixels in various locations
  3. Label some flesh pixels
  4. Label air pixels on the outside of the specimen

The more variety of labeled pixels provided, the better the model will work. However, you don't need to be exhaustive for this approach.

Setting Up the Machine Learning Model [00:04:32]

  1. Navigate to AI > Machine Learning Segmentation
  2. Create a new model:
  3. Name it "Demo on chameleon"
  4. Set the input dataset (the chameleon scan)
  5. Set the sparse data as output

  6. No mask is needed for this particular tool

  7. Select an algorithm:

  8. Extra Trees is recommended as a good option

  9. Choose features to use:

  10. Activation map is particularly effective
  11. You can select multiple features, but for this demo we'll only use one

Training the Model [00:05:37]

Once all parameters are set, start the training process. Training time will vary depending on your data and computer specifications. This is a good time to take a break while the model trains.

Reviewing Results [00:07:53]

After training completes:

  1. The model automatically applies to the dataset
  2. Review the segmentation results
  3. Even with minimal labeling, the model produces reasonable results
  4. Some excess material may be segmented as bone in this quick demo

If you're satisfied with the results, you can apply the model to the entire dataset to create a complete multi-ROI result across all slices.

Conclusion [00:08:52]

This traditional machine learning segmentation approach offers a fast and simple method for segmentation tasks. While not perfect, it can produce good results with minimal manual labeling. For more advanced segmentation, the Segmentation Wizard tool provides an alternative approach using sparse data labeling, which will be covered in a subsequent tutorial.
