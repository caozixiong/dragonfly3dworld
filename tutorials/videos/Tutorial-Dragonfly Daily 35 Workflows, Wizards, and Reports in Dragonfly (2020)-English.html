Workflows, Wizards, and Reports in Dragonfly 2020.1

Introduction [00:00:00]

Welcome to the Dragonfly Daily. This is Lesson 35, focusing on workflows, wizards, and reports in Dragonfly 2020.1. This tutorial will demonstrate how to use Dragonfly's reporting engine and how workflows are enabled through the wizards infrastructure.

The Reports Engine [00:01:04]

Dragonfly has developed a reporting engine to make it easy to capture information from your analysis in a standardized and user-friendly way. The reporting engine allows:

  • Customizable report templates
  • Capturing scalar values (e.g., trabecular thickness in bone analysis)
  • Including tables of measurements
  • Adding histograms, plots, and screenshots
  • Flexible section organization based on analysis needs

The reporting system is designed to be adaptable to different analysis scenarios. For example, when doing bone analysis, you might include an anisotropy section in one report but only trabecular thickness and spacing in another.

XML Data Structure

The reporting engine uses an XML data structure that is:
- Flexible and extensible
- Capable of encoding all report data
- Programmatically accessible

The workflow is:
1. Dragonfly extensions and plugins produce an XML data structure containing report data
2. The XML data is transformed into a PDF using XSL (Extensible Style Sheet Language) Transforms

This approach provides several advantages:
- You can generate many reports over time and store them as XML
- Report formatting can be changed later by modifying the style sheet without regenerating the data
- XML data can be repurposed for other uses (databases, HTML websites, etc.)

Report Generator Tool

In Dragonfly 2020.1, you can find the Report Generator under the Utilities menu. This simple tool allows you to:
- Specify an XML file
- Specify an XSLT transform
- Input key information about the report
- Set the output file name
- Generate the report

Bone Analysis Wizard Example [00:07:08]

Let's demonstrate the reporting system using the Bone Analysis Wizard:

Step 1: Segment the Bone Image [00:08:47]

  1. Open the Bone Analysis Wizard by right-clicking on a bone sample and selecting "Bone Analysis Wizard"
  2. Create an initial segmentation using Define Range Upper O2
  3. Add the segmentation to the selected bone ROI
  4. Fill any gaps in the bone surface by specifying the maximum hole size (e.g., 150 microns)
  5. Click "Fill" to process the segmentation

The wizard performs several operations including:
- Distance map operations
- Closing operations
- Hole filling operations

Step 2: Segment Cortical and Trabecular Bone [00:11:49]

  1. Choose a segmentation method (Bowie or Kohler)
  2. Set the trabecular thickness parameter (e.g., 55 microns)
  3. Click "Segment" to separate the bone into cortical and trabecular regions

The segmentation follows published algorithms that use mathematical morphology operations to identify which pixels should be labeled as cortical and which as trabecular.

Step 3: Global Measurements [00:15:57]

  1. Select the measurements you want to compute (e.g., trabecular thickness, BVTV)
  2. Click "Compute Measurements"
  3. View the results in the table

Step 4: Compute Scalar and Vector Fields [00:17:17]

This step allows you to compute additional measurements like anisotropy and mapping measurements.

Step 5: Generate the Report [00:17:31]

  1. Enter project information:
  2. Project title (e.g., "Drug X Study")
  3. Sample identifier (e.g., "Control Bone 33")

  4. Description of the analysis

  5. Add screenshots to include in the report:

  6. Drag views to the Multiple Screenshots panel

  7. Add metadata and captions for each screenshot

  8. Click "Generate" to create the report

Report Output [00:19:59]

The generated PDF report includes:
- Logo and header with project information
- Title, project name, job ID, date, and authorship
- Global measurements in a table format
- Screenshots with captions and descriptions

The report format is customizable by modifying the XSLT transform. The underlying XML file contains all the data and references to screenshots, making it easily accessible for other programs or for aggregating data across multiple reports.

Extending the Reporting Framework [00:22:37]

The current implementation allows Dragonfly to generate XML files and transcode them to PDFs. Users can:
- Write their own code to process XML files
- Create custom reporting plugins
- Aggregate data from multiple reports
- Convert reports to other formats like HTML

Currently, the reporting engine has built-in support for bone analysis and plug analysis, but the framework can be extended to other applications through custom extensions.

Conclusion

The Dragonfly reporting engine provides a flexible framework for capturing and presenting analysis results. Combined with wizards that guide users through workflows, it creates a powerful system for standardizing analysis procedures and documenting results. The XML-based approach ensures that reports can be easily customized and data can be repurposed for various needs.
